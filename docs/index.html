Getting started

At the bare minimum, Skin simply requires you to reference it's stylesheet in the <head> of each page you plan on using it.

Self-hosted

You can self-host Skin by downloading Skin on GitHub and uploading the files to your web server. Then, reference the location of bfm-skin.min.css.


							<link rel="stylesheet" href="css/bfm-skin.min.css" type="text/css" media="all" />

CDN delivery

You can utilize Bayfront Media's CDN to use Skin without having to download any files at all by referencing the following file:


							<link rel="stylesheet" href="//s1.bayfrontcdn.com/skin/latest/css/bfm-skin.min.css" type="text/css" media="all" />

This file will automatically be kept up to date with the latest version of Skin.

Once you've included this file, you can start building some pages!

What it is

The file bfm-skin.min.css is a minimized group of multiple stylesheets included with Skin. These are (in order):

File Description
vendor/bootstrap/bootstrap-reboot.min.css Bootstrap's method of "normalizing" our environment before we start customizing it with our own styles
css/bfm-grid-skeleton.css This is our own modified and scaled-down grid based heavily on Bootstrap's version
css/bfm-environment.css This is a collection of all the environment styling for Skin
css/bfm-customize.css This is a collection of all the customizable styling classes for Skin

Customization

You can add and reference your own stylesheet to add to or override any styles that are included with Skin. In this example, we are using style.css for this purpose.
